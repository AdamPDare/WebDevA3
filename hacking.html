    Protection mechanisms against hacking

    Encryption

    Encryption is the method of converting plain text into cipher text, using an algorithm. The algorithm will scramble/ encrypt the plain text/ data using a unique key. The only way a recipient can read/ view the true contents of the file or text, they will have to use the key to decrypt it.

    To encrypt a data file, it is done by several bits for each section that is encrypted. The maximum number of bits that can be used is 256-bits and that gives 2^256 possibilities.

    There are various encryption techniques to secure the contents of a file or make some text unreadable to any unauthorised people. The most common Encryption methods are:

    • Advanced Encryption Standard: This type of encryption uses an algorithm that will encrypt sections of data (128 bits) at a time. This type of encryption can be decrypted via a 128,192 or a 256-bit key.

    • Rivest Shamir Adleman: The RSA uses an algorithm that revolves around the factorisation of the products from two large prime numbers. This type of encryption can only be decrypted via knowing these two prime numbers.

    • Triple Data Encryption Standard: Normal Data Encryption Standard uses a 56-bit key to encrypt data. TDES uses DES but encrypts each section of the data three times.

    • Twofish: A licence free encryption technique that will encrypt the data sections via 128-bit encryption.
